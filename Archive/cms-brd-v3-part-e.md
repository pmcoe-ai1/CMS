### Part E: Widgets & Extensibility

The CMS must provide the following categories of user interaction widgets for use on published websites. These widgets are the interactive elements that website visitors see and interact with. The CMS must be able to use existing plugins from WordPress, Umbraco, and other major CMS platforms to deliver these widgets, and must also be able to create new custom widgets when existing plugins do not meet requirements. Widget availability is tenant-scoped — each tenant's widget library is managed independently (per 5.9.1).
